Learning cycle - intelligence layer

Executive summary: The refactor cycle succeeded in establishing strict entity context controls, but residual hardcoded logic, incomplete secret migration, and legacy table gaps remain in every stack. QA coverage fell below the mandate, sustaining a known risk pattern and especially impacting COBOL. CI guardrails targeting hardcode elimination, test-count floors, and secret scans are critical to closing compliance and audit gaps. Full secret migration, partitioning, and refined COBOL resourcing will likely yield complete payoff next cycle, saving an estimated 16 hours.

Observations

  • o1 - Residual entity-specific hardcoded logic is present in every stack despite refactor attempts.
    Evidence: Engineer outputs flagged 1 SQL, 1 Java, 2 COBOL, and 2 Python residual hardcodes.
  • o2 - QA test coverage per package remains at 6, below the hardcode-risk floor set by previous cycles.
    Evidence: QA verdicts are 'Pass with notes' in all packages, with tests=6 per package.
  • o3 - Secrets remain exposed in plaintext configuration files, leaving the environment vulnerable.
    Evidence: Plaintext YAML secrets are reported as not yet fully migrated to Key Vault (pitfall documented).
  • o4 - Database compliance gaps exist due to unpartitioned legacy tables, risking regulatory breach.
    Evidence: Legacy/reference tables lack entity_id partitioning (pitfall documented).
  • o5 - Code improvements are not live, delaying actual compliance and audit benefits.
    Evidence: All 4 PRs are in rehearsal; none merged to main.

Patterns detected

  • Residual hardcodes cluster when QA test count is at or below 6 (as in seeded baseline v7 and confirmed this cycle).
  • COBOL tasks consistently present higher residual hardcode counts, aligning with prior cycle risk and effort underestimation.
  • Incomplete Key Vault migration persists at cycle sign-off, echoing repeated next-step recommendations from previous runs.

Recommendations (ranked)

  • P0 Eliminate all flagged entity-specific hardcodes in SQL, Java, COBOL, and Python before merge. (Tech Lead (with stack DEs), ~12h)
    Ensures regulatory compliance and prevents cross-entity leakage; aligns with mandated architecture.
  • P0 Mandate QA minimum 8 test cases per package if hardcodes are detected (enforced via CI test-count check). (QA Lead, ~4h)
    Mitigates recurring risk where low test coverage correlates with undetected hardcoded logic.
  • P1 Complete Key Vault secrets migration and remove plaintext secrets from YAML configs in production. (SecOps, ~8h)
    Reduces credential exposure risk and meets HK/CH security mandates.
  • P1 Partition all remaining legacy/reference tables by entity_id (SQL migration, data backfill). (Database Engineer, ~10h)
    Eliminates residual data residency breach and audit non-conformance risks.
  • P2 Apply 1.5x standard effort buffer for COBOL remediation and regression, per prior underestimation. (EM/Delivery Manager, ~0h)
    Addresses recurring COBOL risk; prevents missed deadlines.

Guardrails for next cycle

  • Must not merge any PR if entity-specific logic or references (e.g., 'SG', 'HK', 'CH') are found in source outside of config/mapping locations.
    Enforced by: CI static scan job: entity-hardcode-check
  • Must require entity_id partitioning in all newly-refactored or legacy-regulated tables.
    Enforced by: DB migration reviewer checklist; enforced via code review template
  • Must not allow deployment if any YAML config in production contains secret values.
    Enforced by: CI scan: yaml-secrets-linter
  • Must enforce minimum QA test count of 8 for any package where hardcode flags are raised.
    Enforced by: CI gate: qa-test-count-check

Risk flags

  • high Residual hardcoded logic in source increases risk of cross-entity data leakage and audit breach.
    Mitigation: Automated static analysis with merge block on detection; urgent DE review before release.
  • medium Plaintext YAML secrets in production raise vulnerability to credential theft.
    Mitigation: Finalise Key Vault migration with CI-enforced pre-deploy secret check.
  • medium Unpartitioned legacy tables remain a compliance and residency audit risk.
    Mitigation: Priority SQL migration and extended regression before enabling new entity provisioning.
